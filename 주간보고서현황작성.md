# 악성 파일 분석 프로젝트 주차별 개발 진척 현황

## 1~2주차: 기본 분석 환경 및 코어 모듈 구축
- 입출력 인터페이스 및 로깅 환경 구축
- 해시 기반 VirusTotal API 연동 기능 및 쿼리 질의 자동화
- JSON 데이터 파싱 및 분석 데이터 생성
- 분석 대상 파일 포맷 설정 및 카테고리별(PDF, Office, Image, Executable 등) 파일 구조 분석 로직 구현

---

## 3주차: 대규모 파일 병렬 처리, 심층 분석 및 AI 정밀 분석 모듈 연동
현재까지 새롭게 개발 및 고도화된 3주차 주요 실적은 다음과 같습니다.

### 1. 생성형 AI 기반 종합 보안 분석 자동화 도입 (`ai_analysis.py`)
- `gemini-cli` (또는 기타 LLM 도구)를 파이썬 스크립트 내에 연동하여 정밀 분석 단계 도입
- 추출된 파일 정보, 악성 문자열, 백신 진단 결과(VT) 등을 통합하여 AI 모델에 지시(Prompt)하는 로직 개발
- 분석 파일명, 해시, 파일 정보 등을 토대로 악성/정상 여부 확률을 판별하고 최종 권고사항을 포함하는 마크다운(.md) 형태의 종합 보고서 자동 생성

### 2. 비동기(Async) 및 멀티프로세싱을 통한 대량 파일 분석 성능 향상
- `eml_analysis.py` 및 `ai_analysis.py` 내에 `asyncio` 모듈을 전면 도입하여 대량의 파일 스캔 시 I/O 병목 현상 해소
- `concurrent.futures.ProcessPoolExecutor`를 활용하여 복수 첨부파일에 대한 분석 연산을 백그라운드 워커에 분배해 병렬(Parallel) I/O 가속 구현
- 지정된 폴더(예: `./eml`, `./eml_bank`) 내의 다수 파일을 동시에 읽고 분석하는 일괄 처리(Batch Processing) 아키텍처 완성

### 3. 복합 파일(EML 등) 추출 파이프라인 및 연계 처리 최적화 (`eml_analysis.py`)
- EML 이메일 파일을 분석하기 위해 첨부파일, 본문 URL, 메일 헤더, 평판(SPF, DKIM 등)을 독립적으로 분리하여 추출하는 파이프라인(`extract_eml` 모듈 연계) 확립
- 이메일 내에서 추출된 여러 의심스러운 첨부 파일들을 `file_analysis.py`와 연동하여 자동으로 후속 분석을 진행하는 체인(Chain) 로직 구성
- 단일/일괄 EML의 분석 결과를 하나의 텍스트(`_summary.txt`)로 전처리하여 한눈에 파악할 수 있도록 데이터 구조화

### 4. 문서형 악성 파일(Office 문서) Deep Analysis 모듈 구현 (`file_analysis.py`)
- DOCX, XLSX 등 최근 악성코드 유포에 자주 사용되는 OpenXML(ZIP 형태) 기반 문서들에 대한 심층 분석(`deep_analysis`) 기능 추가
- 내부 압축을 해제하여 숨겨진 매크로 바이너리(`vbaProject.bin`), 임베드된 객체 탐지 기능 구현
- `grep`, `strings` 및 전문 도구(`olevba`, `peepdf`)를 파이프라이닝하여 쉘 스크립트(Powershell, CMD 등)나 외부 접근 악성 URL(http, 서드파티 링크)을 스캐닝하는 정밀 탐지 기능 고도화
